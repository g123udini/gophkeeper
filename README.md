# GophKeeper

**GophKeeper** — это современная клиент-серверная система для надежного и безопасного хранения ваших приватных данных: логинов, паролей, банковских карт, текстовых заметок, бинарных файлов и другой конфиденциальной информации.

---

## Основные возможности

- **регистрация, аутентификация и авторизация пользователей;
- **хранение приватных данных;
- **синхронизация данных между несколькими авторизованными клиентами одного владельца;
- **передача приватных данных владельцу по запросу.


---

## Бинарные файлы

Доступны для linux, windows, macos на странице с релизами

## Разработка

```shell script
make build      # Сборка образа
make up         # Запуск серверной части
make logs       # Посмотреть логи
```

# GophKeeper CLI Client

CLI-клиент для работы с GophKeeper через gRPC.

Позволяет:
- регистрироваться
- авторизовываться
- работать с пользовательскими данными
- синхронизироваться с сервером

---

## 🚀 Запуск

```bash
go run ./cmd/client [options] <db_path> <master_password>
```

### Пример

```bash
go run ./cmd/client -addr 127.0.0.1:50051 ./var/client.db my-master-password
```

### Параметры

| Флаг        | Описание                         | По умолчанию      |
|------------|----------------------------------|-------------------|
| `-addr`     | адрес gRPC сервера              | `localhost:50501` |
| `-interval` | интервал синхронизации (сек)    | `60`              |

---

## 🔐 Концепция безопасности

Клиент использует **два уровня защиты**:

### 1. Пароль пользователя
Используется для авторизации на сервере:

login / password

### 2. Master password
Передаётся при запуске клиента:

```bash
go run ./cmd/client ... <master_password>
```

Используется для:
- локального шифрования данных
- защиты клиентской БД

---

## 👤 Регистрация

Создаёт нового пользователя на сервере.

### Команда

register <login> <password>

### Пример

register kirill qwerty123

### Что происходит

- клиент отправляет gRPC-запрос
- сервер создаёт пользователя
- сервер возвращает токен
- клиент сохраняет токен

---

## 🔑 Логин

Авторизация пользователя.

### Команда

login <login> <password>

### Пример

login kirill qwerty123

---

## ⚠️ Важно

Если не выполнить login, защищённые методы работать не будут.

Ошибка:

rpc error: code = Unauthenticated desc = authorization token is missing

---

## 🔄 Синхронизация

После запуска клиент автоматически:

- запускает фоновую синхронизацию
- периодически обращается к серверу

Если не выполнен login — будет warning:

sync: can`t get updates: authorization token is missing

---
